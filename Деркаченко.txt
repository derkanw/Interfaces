1. Класс TextEdit хранит указатель на объект класса QTextEdit, являясь для последнего интерфейсной оболочкой. То есть методом TextEdit setCentralWidget(textEdit) экземпляр QTextEdit устанавливается центральным виджетом, на котором находится фокус.
За состояние ползунка отвечает класс QScrollBar, экземпляр которого содержится в QTextEdit. А класс QScrollBar унаследован от класса QAbstractSlider.
Аналогом сообщения WM_VSCROLL являются сигналы QAbstractSlider::actionTriggered и QAbstractSlider::sliderPosition, которые обрабатываются методом void QTextEdit::scrollContentsBy(int dx, int dy). Этот метод представляет собой переопределение виртуального метода класса QAbstractScrollArea. Метод вызывается для перемещения содержимого области при изменении состояния QScrollBar.
Параметры dx и dy используются для удобства, чтобы указать, насколько именно следует прокрутить (полезно, например, при выполнении сдвига пикселей). Но можно и игнорировать эти значения и прокручивать непосредственно в положение, указанное полосами прокрутки.

2. Узел "DOM-дерева" является подклассом QDomNode и содержит следующие атрибуты:
а) enum QDomNode::EncodingPolicy - определяет, откуда будут сериализованы данные для сохранения методом QDomNode::save() в "DOM-дерево": из документа или из QTexeStream.
б) enum QDomNode::NodeType - определяет тип узла, то есть какой вид данных будет храниться в этом узле. Например, QDomNode::AttributeNode, QDomNode::DocumentNode, QDomNode::CharacterDataNode и т.д.

3. Выгруженный текст хранится в классе QTextDocument в виде QString и имеет две структуры, представленные деревьями, основанными на "DOM-дереве": QFragmentMap<QTextBlockData> BlockMap и QFragmentMap<QTextFragmentData> FragmentMap. То есть текст делится на параграфы (QTextBlock), которые содержат в себе фрагменты форматированного текста (QTextFragment), каждый из которых имеет свой стиль и формат, например, шрифт и размер.
Следовательно, BlockMap группирует в себе все параграфы документа с разными стилями и используется для представления абзацев в документе. А FragmentMap - фрагменты параграфа, которые добавляются при редактировании документа. Фрагменты могут объединяться и удаляться для эффективного представления различных стилей текста в блоке.
Эти структуры используются для более эффективного представления текста, вставки и удаления символов.
